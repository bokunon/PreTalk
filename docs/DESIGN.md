# PreTalk 設計書（ドラフト）

## 1. 概要・目的

### 背景・課題
商談アポ取得時点では、**会社名・相手の部署・名前**程度しか情報がない。初回商談で何を話すか毎回悩む。

### 現状の手作業
- IR情報、理念・ビジョンの調査
- 担当者・部署のインタビュー記事検索
- 関連ニュースの収集
- 中期経営計画・ビジョン → 当該部署のやるべきことの推測
- できてなさそうなこと・困ってそうなことの特定
- 提案ネタの探索

### 目的
上記の下調べを自動化し、初回商談用のトピック・ネタをまとめて出力するツールを提供する。

---

## 2. 入力・出力

### 入力（自然言語）
| 項目 | 説明 | 例 |
|------|------|-----|
| 会社名 | 商談相手の企業名 | 株式会社サンプル |
| 部署名 | 相手の所属部署 | 経営企画部 |
| 担当者名 | 相手の氏名（任意） | 山田太郎 |

### 出力
- **会社概要**: 事業内容、理念・ビジョン、中期経営計画の要約
- **部署の文脈**: 当該部署の役割、最近の動き、インタビュー等からの情報
- **課題・ニーズの推測**: できてなさそうなこと、困ってそうなこと
- **提案ネタ**: 初回で持っていくと良さそうな話題・アイデア
- **参考リンク**: 情報元のURL一覧

## 3. 処理フロー

```
[入力: 会社名, 部署名, 担当者名]
        │
        ▼
┌───────────────────────────────────────┐
│ Phase 1: 検索クエリ生成                 │
│ ・会社IR、理念、ビジョン、中期経営計画   │
│ ・部署名 + インタビュー、ニュース        │
│ ・担当者名 + 記事（あれば）              │
└───────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────┐
│ Phase 2: Web検索実行                    │
│ ・各クエリで検索 → 上位N件のURL取得     │
└───────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────┐
│ Phase 3: コンテンツ取得・要約            │
│ ・URLから本文を取得（スクレイピング）    │
│ ・テキストの要約・構造化                │
└───────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────┐
│ Phase 4: AI分析・統合                    │
│ ・収集情報をLLMに渡す                   │
│ ・課題推測・提案ネタの生成              │
│ ・初回商談トピックの構造化出力          │
└───────────────────────────────────────┘
        │
        ▼
[出力: 商談トピックまとめ + 参考リンク]
```

## 5. データソース・技術選定

| 用途 | 候補 | 初回ドラフトでの採用 |
|------|------|---------------------|
| Web検索 | SerpAPI, Brave Search, DuckDuckGo | Brave Search API（無料枠あり）またはDuckDuckGo |
| コンテンツ取得 | Puppeteer, Cheerio, Readability | Cheerio + 簡易本文抽出 |
| LLM | OpenAI, Anthropic, ローカルLLM | OpenAI API（環境変数でキー指定） |

### 環境変数
- `OPENAI_API_KEY`: LLM分析用
- `BRAVE_SEARCH_API_KEY`: Web検索用（任意、なければDuckDuckGo等の代替）
